## このアプリでできること

**打ち上げ調整くん**は、複数人での予定をかんたんに調整できるWebアプリです。  
GitHubアカウントでログインし、イベントの候補日程を作成して、参加・不参加を入力できます。

- 予定の作成／編集／削除  
- 複数の候補日程をまとめて登録  
- 各日程ごとの参加・不参加の登録  
- コメントの投稿  
- 出欠やコメントは画面をリロードせずに即時反映  

「LINE交換をせずに、あとから集まる予定を決めたい」  
そんな場面を想定して作っています。

## 開発した背景

このアプリを作る前に、**ライブ同行者を募集する掲示板アプリ**を個人開発しました。

### 🎤 ライブ同行掲示板
https://livemate.vercel.app/

その掲示板を通して、

- SNS上で知り合った人と  
- 匿名に近い形で  
- 一緒にライブへ行く  

という体験はできるようになりました。

一方で、ライブ後の打ち上げやオフ会については、

- その場で全員とLINEを交換するのは気が引ける  
- 誰が来られるのか分からず、日程調整がグダグダになりやすい  

といった課題があると感じました。

そこで、  
**「ライブに一緒に行った人たちと、LINE交換をせずにあとから予定を決められるアプリがあればいい」**  
と思い、この「打ち上げ調整くん」を開発しました。

## 使っている技術

- **Node.js + Hono**  
  サーバー側の処理を実装。Honoは軽量でシンプルなWebフレームワーク。

- **PostgreSQL + Prisma**  
  予定・日程・出欠などのデータを保存するためのデータベースと、その操作を分かりやすく書くためのORM。

- **GitHubログイン**  
  GitHubアカウントを使った認証を実装。

- **Bootstrap + JavaScript**  
  画面の見た目と、出欠・コメントの非同期更新に使用。

- **Zod**  
  フォームやAPIで受け取った値のバリデーションに使用。

---

## 画面まわりの工夫

- 出欠の変更やコメント投稿はページを再読み込みせずに反映  
- 候補日程は複数行入力から自動で分割して生成  
- 予定一覧は更新日時が新しいもの順に表示  

---

## データベース

- 予定  
- 候補日程  
- 出欠情報（ユーザー × 候補日程）  
- コメント  

これらを分けて管理しています。

同じユーザーが同じ予定に対して何度操作しても、  
データが壊れないような設計をしました。

---

## セキュリティ面で気をつけたこと

- ログインしていないユーザーは操作できない  
- 他人の出欠やコメントを勝手に変更できない  
- 不正なリクエストを防ぐための基本的な対策を実装
